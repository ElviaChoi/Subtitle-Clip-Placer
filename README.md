# 🎬 Subtitle Clip Placer

SRT 자막의 시작 시간에 맞춰 짧은 영상 클립을 자동으로 이어 붙이는 Windows용 GUI 도구입니다.

Vrew 등에서 만든 SRT와 Grok 같은 생성형 영상 도구로 만든 클립들을 연결해, 대사 흐름에 맞는 최종 MP4를 빠르게 만들 수 있습니다.

## ✨ 주요 기능

- SRT 자막 시작 시간을 기준으로 영상 슬롯 자동 생성
- 영상 파일명 앞번호(`001_`, `002_` 등)를 기준으로 자동 매칭
- Excel에서 수정 가능한 CSV 작업표 생성
- AI 장면표의 `한글 내레이션` 열을 SRT에 자동 매칭
- AI 장면표의 `시작~끝` 시간과 `시작 자막`을 함께 사용해 더 정확하게 매칭
- 빈 `영상파일` 칸은 다음 영상 시작 전까지 직전 영상 유지
- 영상 시작 번호 지정으로 같은 폴더의 `015_`번 영상부터 이어 작업 가능
- 작업 시작/종료 번호 지정으로 전체 SRT 중 일부 구간만 생성 가능
- 포함된 `ffmpeg/bin/ffmpeg.exe` 자동 인식
- 최종 영상에서 원본 영상 소리 제거
- 생성 중 `중지` 버튼으로 현재 FFmpeg 작업 중단 가능

## 🚀 사용 흐름

1. Vrew 등에서 자동 자막을 만든 뒤 SRT로 내보냅니다.
2. 생성한 영상 파일들을 한 폴더에 넣고 `001_`, `002_`, `003_`처럼 앞번호를 붙입니다.
3. `run_gui.bat`을 더블클릭합니다.
4. `SRT 파일`, `영상 폴더`, `저장 위치/파일명`을 선택합니다.
5. 필요하면 `작업 시작 번호`와 `작업 종료 번호`로 이번에 만들 구간을 지정합니다.
6. 필요하면 `AI 장면표로 매핑` 또는 `Excel 작업표 만들기`로 CSV 매핑을 만듭니다.
7. `대사 기준 배치 미리보기`에서 연결 상태를 확인합니다.
8. `최종 영상 생성`을 누릅니다.

## 🧩 AI 장면표 매핑

AI에게 장면표를 만들 때 `시작~끝` 열과 `시작 자막` 또는 `한글 내레이션` 열에 각 영상이 시작될 대사 일부를 적게 하면 됩니다.

예시:

```text
#	길이	장면	한글 내레이션	화면 컨셉
1	00:00~00:05 (5초)	A	첫 번째 장면의 시작 대사	도입 장면
2	00:05~00:11 (6초)	B	두 번째 장면의 시작 대사	강조 장면
3	00:11~00:19 (8초)	C	세 번째 장면의 시작 대사	설명 장면
```

프로그램은 `#` 번호가 아니라 `시작~끝` 시간 근처에서 `시작 자막` 또는 `한글 내레이션` 문구가 가장 잘 맞는 SRT 시작 지점을 찾습니다.

`영상 시작 번호`를 `15`로 설정하면 장면표 1번은 `015_...mp4`, 장면표 2번은 `016_...mp4`부터 연결됩니다.

SRT 문장이 여러 줄로 나뉘거나 살짝 다르게 표현된 경우도 인접 자막과 유사도 매칭으로 최대한 찾습니다.

## ✂️ 부분 작업

전체 SRT를 넣은 상태에서도 `작업 시작 번호`와 `작업 종료 번호`로 이번에 만들 구간만 지정할 수 있습니다.

예를 들어 CSV 맨 왼쪽 번호 기준으로 `182`번부터 `199`번까지만 만들고 싶다면:

```text
작업 시작 번호: 182
작업 종료 번호: 199
```

종료 번호 뒤의 자막은 이번 출력에서 제외됩니다. 뒤쪽을 이어서 만들 때는 다음 시작 번호부터 다시 생성하면 됩니다.

## 📄 CSV 작업표

CSV 열은 아래와 같습니다.

```text
번호,시작시간,길이초,대사,작업,영상파일
```

Excel에서는 보통 `작업`과 `영상파일` 칸만 확인하거나 수정하면 됩니다.

```text
1,00:00:01.200,3.100,안녕하세요 오늘은...,영상,001_grok-video.mp4
2,00:00:04.300,4.000,다음 장면입니다...,이전유지,
3,00:00:08.300,2.500,잠깐 쉬어갑니다...,검은화면,
```

`작업` 열에는 아래 값을 쓸 수 있습니다.

```text
영상: 이 대사 시작점에 새 영상을 넣습니다.
이전유지: 앞 대사에서 시작한 영상을 계속 보여줍니다.
검은화면: 이 대사 구간은 영상 없이 검은 화면으로 둡니다.
```

`작업=영상`인데 `영상파일` 칸이 비어 있으면 `이전유지`처럼 처리합니다. 즉, 새 영상 파일명이 나올 때까지 직전 영상이 계속 재생됩니다.

## ⚙️ 처리 옵션

- `부족한 영상 처리` 기본값은 `반복 후 자르기`입니다.
- `반복 후 자르기`: 영상이 슬롯보다 짧으면 처음부터 반복 재생하고 필요한 길이에서 자릅니다.
- `느리게 늘리기`: 영상이 슬롯보다 짧으면 필요한 길이에 맞춰 느리게 늘립니다.
- `자동`: 영상이 조금 짧으면 느리게 늘리고, 많이 짧으면 반복 후 자릅니다.
- `느리게 허용 배율` 기본값은 `1.2`입니다. 원본 길이의 120%까지는 느리게 늘립니다.
- `마지막 자막 길이` 기본값은 `8.0`초입니다.

## 🎞️ 영상 파일명

권장 파일명은 아래 형식입니다.

```text
001_영상제목.mp4
002_영상제목.mp4
003_영상제목.mp4
```

뒤쪽 제목은 길거나 달라도 괜찮습니다. 앞 숫자만 맞으면 됩니다.

## 🛠️ FFmpeg

이 프로그램은 내부적으로 FFmpeg를 사용합니다.

폴더 안에 `ffmpeg/bin/ffmpeg.exe`가 있으면 GUI가 자동으로 경로를 잡습니다.

직접 지정해야 한다면 GUI의 `ffmpeg.exe` 칸에서 선택하세요. 같은 폴더에 `ffprobe.exe`도 있어야 합니다.

## ▶️ 실행

Windows에서:

```powershell
run_gui.bat
```

또는 Python으로 직접 실행:

```powershell
python subtitle_clip_placer.py
```
