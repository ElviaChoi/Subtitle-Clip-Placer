# Subtitle Clip Placer

SRT 자막의 시작 시간에 맞춰 영상 파일을 자동으로 이어 붙이는 Windows용 GUI 프로토타입입니다.

## 사용 흐름

1. Vrew에서 자동 자막을 만든 뒤 SRT로 내보냅니다.
2. Grok 등에서 만든 짧은 영상들을 한 폴더에 넣습니다.
3. 영상 파일 이름 앞에 자막 순서 번호를 붙입니다. 예: `01_grok-video.mp4`, `02_grok-video.mp4`, `03_grok-video.mp4`
4. `run_gui.bat`을 더블클릭합니다.
5. SRT 파일, 영상 폴더, 저장 위치를 선택합니다.
6. `대사/영상 미리보기`에서 각 대사에 어떤 영상이 붙는지 확인하고 `생성하기`를 누릅니다.
7. Excel에서 확인하고 싶으면 `CSV 작업표 저장`을 누른 뒤 CSV를 열어봅니다.

## 처리 규칙

- 각 자막 시작점과 다음 자막 시작점 사이를 하나의 영상 슬롯으로 봅니다.
- 화면에는 자막 대사 기준으로 배치 미리보기를 보여줍니다.
- 영상 파일명 앞 숫자가 있으면 그 숫자를 기준으로 대사 번호에 연결합니다.
- 영상이 슬롯보다 길면 슬롯 길이만큼 자릅니다.
- 자동 모드에서 영상이 조금 짧으면 느리게 늘립니다.
- 자동 모드에서 영상이 많이 짧으면 반복 재생 후 넘치는 부분을 자릅니다.
- 기본 자동 기준은 `1.2`입니다. 예를 들어 6초 영상은 7.2초 슬롯까지 느리게 늘리고, 그보다 긴 슬롯은 반복합니다.
- 마지막 자막은 기본 6초 슬롯으로 처리합니다.
- 최종 영상에는 원본 영상 소리를 넣지 않습니다.

## CSV 작업표

SRT를 직접 적을 필요는 없습니다. Vrew에서 내보낸 SRT를 선택한 뒤 `CSV 작업표 저장`을 누르면 Excel에서 열 수 있는 CSV가 만들어집니다.

구조를 먼저 보고 싶으면 이 폴더의 `sample_mapping.csv`를 Excel로 열어보세요.

CSV 열은 아래와 같습니다.

```text
번호,시작시간,길이초,대사,작업,영상파일
```

Excel에서는 보통 `작업`과 `영상파일` 칸만 확인하거나 바꾸면 됩니다.

```text
1,00:00:01.200,3.100,안녕하세요 오늘은...,영상,01_grok-video.mp4
2,00:00:04.300,4.000,다음 장면입니다...,이전유지,
3,00:00:08.300,2.500,잠깐 쉬어갑니다...,검은화면,
```

CSV를 수정했다면 GUI의 `CSV 매핑(선택)`에 그 CSV를 선택하세요. 그러면 `영상파일` 열을 기준으로 대사에 영상을 연결합니다.

`작업` 열에는 아래 값을 쓸 수 있습니다.

```text
영상: 이 대사 시작점에 새 영상을 넣습니다. 영상파일 칸이 필요합니다.
이전유지: 앞 대사에서 시작한 영상을 계속 보여줍니다. 영상파일 칸은 비워도 됩니다.
검은화면: 이 대사 구간은 영상 없이 검은 화면으로 둡니다.
```

예를 들어 1번 대사에 `01_scene.mp4`, 2번과 3번 대사에 `이전유지`를 넣으면 `01_scene.mp4`가 1~3번 대사 구간 전체에 이어서 들어갑니다.

## 영상 파일명

권장 파일명은 아래 형식입니다.

```text
01_영상제목.mp4
02_영상제목.mp4
03_영상제목.mp4
```

영상이 100개 이상이면 세 자리 번호가 더 안정적입니다.

```text
001_영상제목.mp4
002_영상제목.mp4
003_영상제목.mp4
```

뒤쪽 제목은 길거나 달라도 괜찮습니다. 앞 숫자만 맞으면 됩니다.

## FFmpeg

이 프로그램은 내부적으로 FFmpeg를 사용합니다.

FFmpeg가 Windows PATH에 등록되어 있으면 `ffmpeg.exe` 칸을 비워도 됩니다.
PATH에 없다면 GUI에서 `ffmpeg.exe` 위치를 직접 선택하세요. 같은 폴더에 `ffprobe.exe`도 있어야 합니다.

## 실행

```powershell
python subtitle_clip_placer.py
```

또는 Windows에서 `run_gui.bat`을 더블클릭합니다.
